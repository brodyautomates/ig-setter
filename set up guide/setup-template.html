Chapter 01 — Overview
// 01
What This Is
An overview of the system, its components, and how they connect.

ig-setter is an open-source dashboard that puts a real-time AI operator inside your Instagram DMs. When someone messages your account, Claude automatically generates a contextual reply. You see the conversation live, can approve the AI draft, or send your own override message — which routes back to Instagram instantly.

This is a private tool for your own account. You control all data. Nothing is shared with third parties beyond the services you explicitly connect.

Architecture

Instagram DM received
↓ Meta Webhook (HTTP POST)
Meta Graph API
↓ triggers
n8n Workflow
↓ calls Claude, writes to DB
Claude AI
→ draft reply →
Supabase DB
↓ realtime subscription
ig-setter Dashboard
↓ override send → n8n → Meta API → reply sent

Prerequisites
Everything you need before starting. Check these off first.
  • Instagram Professional account — Business or Creator. Free to switch in Instagram Settings → Account → Switch to Professional Account.
  • Facebook Page connected to your Instagram. Required by Meta for the Messaging API. Create one free at facebook.com/pages/create.
  • Meta Developer account — free at developers.facebook.com.
  • Supabase account — free at supabase.com.
  • n8n account — free self-host (npx n8n) or cloud at app.n8n.cloud.
  • Anthropic API key — sign up at console.anthropic.com. Add $5 credit to start.
  • Vercel account — free at vercel.com. Connect your GitHub account.
  • Node.js 18+ — only needed for local development. Download at nodejs.org.
⚠
Instagram account type matters Your Instagram account must be a Professional account (Business or Creator) — personal accounts cannot access the Messaging API. This is a Meta platform requirement, not a limitation of ig-setter.

Chapter 03 — Supabase
// 03
Supabase Setup
Create your database and run the migration. Estimated time: 5 minutes.
1
Create a new project

Go to supabase.com → New Project. Name it ig-setter. Choose a region close to your audience. Set a strong database password and save it somewhere safe.

2
Run the migration SQL

In your Supabase project, go to SQL Editor → New Query. Open supabase/migrations/001_init.sql from the repo. Copy and paste the entire file contents into the editor and click Run.

You should see: "Success. No rows returned."

This creates 3 tables (dm_threads, dm_messages, daily_stats), enables Realtime on all three, and configures Row Level Security so the dashboard can read but only the server can write.

3
Copy your credentials

Go to Settings → API. You need three values:

  • Project URL → NEXT_PUBLIC_SUPABASE_URL
  • anon public key → NEXT_PUBLIC_SUPABASE_ANON_KEY
  • service_role secret key → SUPABASE_SERVICE_ROLE_KEY
🔒
Keep the service_role key secret The service_role key bypasses all Row Level Security. Never expose it client-side, never commit it to GitHub, and never share it. It only goes into your Vercel environment variables as a server-side secret.
4
Verify Realtime is enabled

Go to Database → Replication in Supabase. Under "Source" you should see all three tables listed under supabase_realtime. If they're not listed, re-run the migration SQL.

Database Schema

TablePurposeKey Columns
dm_threadsOne row per IG conversationig_thread_id, username, status, pending_ai_draft
dm_messagesIndividual messages in each threaddirection, content, is_ai, override
daily_statsDaily aggregated pipeline statsqualified, booked, closed, revenue

Chapter 04 — Meta App & Instagram API
// 04
Meta App Setup
The most involved step. Read through before starting. Estimated time: 20–30 minutes.
ℹ
Do Step 4 (Deploy) first You need a live HTTPS URL to configure the Meta webhook. Complete Chapter 06 (Deploy to Vercel) before coming back to the webhook configuration section here.
1
Create a Meta Developer App

Go to developers.facebook.com → My Apps → Create App. Select Business as the app type. Fill in App Name (ig-setter), contact email, and link it to a Business Account. Click Create App.

2
Add the Instagram product

In your app dashboard, click Add Product and find Instagram. Click Set Up. In the left sidebar, go to Instagram → API setup with Instagram login. Follow the prompts to connect your Instagram Professional account and linked Facebook Page.

3
Generate an access token

Under Instagram → Generate access tokens, click Generate next to your account. Copy this token — it is your INSTAGRAM_ACCESS_TOKEN.

To convert to a long-lived token (recommended — lasts 60 days vs. 1 hour):

GET https://graph.facebook.com/v19.0/oauth/access_token?grant_type=fb_exchange_token&client_id=YOUR_APP_ID&client_secret=YOUR_APP_SECRET&fb_exchange_token=YOUR_SHORT_LIVED_TOKEN
4
Get your Instagram User ID

Open this URL in your browser (replace the token):

https://graph.facebook.com/v19.0/me?fields=id,username&access_token=YOUR_ACCESS_TOKEN

Copy the id value — this is your INSTAGRAM_USER_ID.

5
Configure the webhook

In your Meta app, go to Instagram → Webhooks. Click Add Callback URL and enter:

  • Callback URL: https://your-app.vercel.app/api/webhook
  • Verify Token: your WEBHOOK_VERIFY_TOKEN value

Click Verify and Save. After verification, click Add Subscriptions and check messages. Save.

6
Request App Review (for production)

In Development mode, the webhook only receives DMs from your own account and approved test users. To receive messages from real followers, you need App Review approval for instagram_manage_messages.

Go to App Review → Permissions and Features → Request. Submit a description of your use case (private DM dashboard for account owner), a screencast, and a privacy policy URL. Approval takes 1–5 business days.

⚠
Token expiry Long-lived access tokens expire after 60 days. Set a calendar reminder to refresh your token before it expires, or your webhook will stop receiving messages. Use the token exchange endpoint above to refresh.

Chapter 05 — n8n Workflow
// 05
n8n Setup
Import the pre-built workflow, add credentials, activate. Estimated time: 10 minutes.

Get n8n Running

Option A  n8n Cloud (easiest) — Sign up at app.n8n.cloud. Free trial, then $20/month. No server management required.

Option B  Self-hosted (free) — Run locally:

npx n8n

Or with Docker:

docker run -it --rm --name n8n -p 5678:5678 n8nio/n8n

Runs at http://localhost:5678. For production, deploy to a VPS or use Railway/Render. You need a public HTTPS URL for Meta to reach your webhook.

1
Import the workflow

In n8n: Workflows → Add Workflow → Import from File. Select n8n/ig-setter-workflow.json from the repo. The workflow has 12 nodes pre-configured.

2
Add Credential 1 — Instagram Access Token

Go to Credentials → New Credential → HTTP Query Auth. Name it Instagram Access Token. Parameter name: access_token. Value: your Instagram access token. Save.

3
Add Credential 2 — Anthropic API Key

Go to Credentials → New Credential → HTTP Header Auth. Name it Anthropic API Key. Header name: x-api-key. Value: your Anthropic API key from console.anthropic.com. Save.

4
Set environment variables in n8n

In n8n Cloud: Settings → Environment Variables. Self-hosted: add to your .env file.

  • DASHBOARD_URL → your Vercel deployment URL (e.g. https://your-app.vercel.app)
  • WEBHOOK_SECRET → the same random string you set in your .env.local
5
Assign credentials to nodes

Open the workflow. Click Get IG Username node → Credentials → select Instagram Access Token. Click Claude — Generate Reply → Credentials → select Anthropic API Key. Click Send IG Reply → Credentials → select Instagram Access Token. Save.

6
Copy webhook URLs

Click Webhook — Meta IG DM node → copy Production URL → paste this into Meta Developer Portal webhook callback field (Chapter 04, Step 5).

Click Webhook — Override node → copy Production URL → this is your N8N_OVERRIDE_WEBHOOK_URL env var.

7
Activate the workflow

Click the toggle at the top of the workflow to set it Active. The webhooks are now live.

Chapter 06–07 — Deploy & Test
// 06
Deploy the Dashboard
Get your dashboard live. Estimated time: 5 minutes.

Option A — Vercel (Recommended)

Click the Deploy with Vercel button in the GitHub repo README. Vercel will clone the repo, prompt you for environment variables, and deploy in ~2 minutes. Add all variables from .env.example with your real values.

Alternatively, deploy from the CLI:

npm install -g vercel cd ig-setter vercel --prod

After deploying, go to Vercel Dashboard → Project → Settings → Environment Variables to add your env vars, then redeploy.

Option B — Run Locally

git clone https://github.com/brodyautomates/ig-setter.git cd ig-setter npm install cp .env.example .env.local # Fill in .env.local with your real values npm run dev

Dashboard runs at http://localhost:3000. To expose a public URL for the Meta webhook during testing, use ngrok:

ngrok http 3000

Use the ngrok HTTPS URL as your webhook callback in Meta.

// 07
Test & Go Live
Verify the full pipeline before opening to real followers.
1
Send a test DM

From a second Instagram account (or a test user account added in the Meta Developer Portal), send a DM to your main account. Wait 5–10 seconds.

2
Check the dashboard

Open your deployed URL. The conversation should appear in the left feed in real-time. Click it to see the message and the AI-generated draft in the Override panel.

3
Test the override

Type a custom message in the Override panel and click Send Override. Check your second test account — it should receive the message via Instagram within a few seconds.

4
Check the n8n execution log

In n8n, go to Executions. You should see a successful execution for each DM event. If any node failed, click it to see the error detail.

Environment Variables
Every variable, where to get it, and which are required.
VariableWhere to Get ItRequired
NEXT_PUBLIC_SUPABASE_URL Supabase → Settings → API → Project URL Yes
NEXT_PUBLIC_SUPABASE_ANON_KEY Supabase → Settings → API → anon public key Yes
SUPABASE_SERVICE_ROLE_KEY Supabase → Settings → API → service_role key Yes
WEBHOOK_SECRET Make up a long random string (20+ chars) Yes
WEBHOOK_VERIFY_TOKEN Make up a random string — enter same value in Meta portal Yes
INSTAGRAM_ACCESS_TOKEN Meta Developer Portal → Instagram → Generate Token Yes
INSTAGRAM_USER_ID Graph API: /me?fields=id&access_token=YOUR_TOKEN Yes
N8N_OVERRIDE_WEBHOOK_URL n8n → Override webhook node → Production URL Yes
ANTHROPIC_API_KEY console.anthropic.com → API Keys n8n only
ℹ
ANTHROPIC_API_KEY goes in n8n, not Vercel Claude is called from the n8n workflow, not from the Next.js app. Add your Anthropic API key as an HTTP Header Auth credential in n8n (Chapter 05, Step 3). You only need it in Vercel if you add direct server-side AI features later.

Generating Secure Secret Strings

For WEBHOOK_SECRET and WEBHOOK_VERIFY_TOKEN, use a cryptographically random string. Generate one with:

node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

Or use any password manager's random password generator (30+ characters, alphanumeric).

Setting Variables in Vercel

In Vercel: Project → Settings → Environment Variables. Add each variable for Production, Preview, and Development environments. After adding, go to Deployments and click Redeploy for them to take effect.

// 09
Customising the AI
Make Claude sound like you and close for your specific offer.

The AI prompt lives in the n8n workflow inside the Claude — Generate Reply node. Edit the system field to match your voice, offer, and closing style.

Coaching / Consulting

You are a high-ticket sales closer for [NAME]'s coaching business. Qualify leads by asking about their current situation and goal. Move qualified leads toward booking a 30-minute strategy call. Keep replies under 3 sentences. Never use emojis. Be direct.

E-commerce / Product

You are a sales rep for [BRAND]. Answer product questions clearly. Offer the payment link when they're ready to buy. Upsell to bundles when appropriate. Keep it warm and fast.

Agency / B2B

You are a BD rep for [AGENCY]. Qualify leads by asking about their business, monthly revenue, and main pain point. Your goal is to book a 20-minute discovery call. Use a consultative tone — ask questions before pitching.
◈
The AI never sends automatically The current workflow generates a draft that appears in your dashboard. You control when messages go out — either by clicking Send Override or by building an auto-send step into your n8n workflow for threads you trust fully.
// 10
Troubleshooting
The most common issues and how to fix them.
Webhook verification fails in Meta portal
Check that WEBHOOK_VERIFY_TOKEN in your Vercel env vars exactly matches what you entered in Meta → Webhooks → Verify Token. Ensure your Vercel deployment is live (not still building) before clicking Verify.
DMs appear in n8n but not on the dashboard
WEBHOOK_SECRET mismatch between Vercel and n8n. Verify both are identical. Also check DASHBOARD_URL in n8n has no trailing slash. Check Vercel → Functions → /api/webhook logs for the specific error.
Yellow setup banner shows on the dashboard
One or more env vars are missing. Click the link in the banner. Make sure all three Supabase vars are in Vercel (especially SUPABASE_SERVICE_ROLE_KEY). Redeploy after adding any missing variables.
n8n fails on "Get IG Username" node
Your access token has expired. Regenerate it in Meta Developer Portal → Instagram → Generate Token. Long-lived tokens last 60 days — use the token exchange endpoint to refresh before expiry.
Override button shows an error
Verify N8N_OVERRIDE_WEBHOOK_URL is the Production URL (not Test URL) from n8n. Make sure the Override webhook node is active in n8n.
No DMs in Development mode from real followers
This is expected. In Development mode, Meta only sends events for your own account and approved test users. To receive from real followers, complete App Review (Chapter 04, Step 6).

Regulations & Compliance
Your legal obligations when operating this system. Read carefully before going live.
⚠
This is informational — not legal advice This chapter outlines the key regulatory frameworks that apply to operating ig-setter. For advice specific to your jurisdiction and use case, consult a qualified legal professional.

1. Meta Platform Terms of Service

Platform Usage

By connecting ig-setter to the Meta Graph API, you agree to Meta's Platform Terms (developers.facebook.com/terms) and the Instagram Platform Policy. Key obligations:

  • Use the API only for its stated purpose — managing messages for your own account
  • Do not use the API to send unsolicited messages (spam)
  • Do not scrape, store, or sell user data beyond what is needed to operate the service
  • Do not use automated messaging to deceive users or impersonate a human when directly asked
  • Comply with Meta's data deletion requirements — if a user requests deletion, delete their data from your Supabase instance
Messaging Restrictions

The Instagram Messaging API has specific rules about when you can send messages:

  • 24-hour window: You can only send messages in response to an inbound message within a 24-hour window. ig-setter is designed for responsive replies, which keeps you within this window.
  • No cold outreach: You may not initiate conversations with users who have not first messaged you. ig-setter only handles inbound DMs — it does not initiate conversations.
  • No promotional messages outside the window: After 24 hours of inactivity, you cannot send promotional content. Transactional/customer service messages may be permitted in some cases.
App Review Requirement

Operating ig-setter with real followers requires instagram_manage_messages permission, which requires Meta App Review approval. Running in production without App Review approval (i.e., using Development mode to receive messages from non-test users) violates Meta's Platform Terms and may result in app suspension.

2. Data Protection & Privacy

GDPR (European Union / UK)

If any of your Instagram followers are based in the EU or UK, the GDPR applies to how you handle their personal data. Key requirements:

  • Legal basis: You must have a lawful basis for processing DM data. For a business responding to customer inquiries, this is typically "legitimate interests" or "contract performance."
  • Privacy notice: Users should be able to find out how their data is used. Add a privacy policy to your Instagram bio or link tree.
  • Data minimisation: ig-setter only stores what is needed — usernames, message content, and timestamps. Do not add fields you don't need.
  • Right to erasure: If a user requests deletion, you must delete their data from Supabase. Do this via the Supabase Table Editor.
  • Data processor agreements: Supabase, Vercel, and Anthropic act as data processors. Each has a Data Processing Agreement (DPA) available — sign these if required in your jurisdiction.
CCPA (California, USA)

If you have followers in California, the CCPA applies. Key obligations:

  • Disclose what personal information you collect and why
  • Do not sell personal information collected via DMs
  • Honour deletion requests within 45 days
CASL (Canada)

Canada's Anti-Spam Legislation applies to commercial electronic messages. Since ig-setter only responds to inbound DMs (it does not initiate), the reply exemption typically applies. However, ensure any outbound messages you send contain no deceptive content and comply with CASL's content requirements.

3. Disclosure of AI

AI Disclosure Obligations

Multiple jurisdictions are introducing or have introduced requirements to disclose AI use in customer communications:

  • EU AI Act (2024): Systems interacting with humans must disclose that they are AI when directly asked. If a user asks "Am I talking to a bot?", your AI must answer truthfully — or your override should be used to respond manually.
  • FTC Guidelines (USA): The FTC has issued guidance that deceptive AI impersonation of humans may constitute an unfair or deceptive trade practice.
  • Best practice: Add a note to your Instagram bio indicating that initial DM responses may be AI-assisted, or configure your AI prompt to acknowledge AI assistance if directly asked.

Recommended prompt addition:

If a user directly asks whether they're talking to a bot or AI, respond: "Initial replies are AI-assisted — [Name] reviews and takes over for all serious conversations."

4. Anthropic Usage Policies

Claude API Terms

By using the Claude API, you agree to Anthropic's Usage Policies (anthropic.com/legal/aup). Relevant restrictions for this use case:

  • Do not use Claude to generate deceptive content intended to mislead recipients
  • Do not use Claude to generate spam or mass unsolicited messages
  • Do not use Claude outputs to harass individuals
  • Anthropic's policies prohibit using the API for illegal discrimination (e.g., screening users based on protected characteristics)

5. Privacy Policy Requirement

You Need a Privacy Policy

Meta requires a privacy policy URL to complete App Review. Additionally, privacy laws in the EU, UK, USA, Canada, and Australia require you to publish a privacy policy if you collect personal data.

Your privacy policy should disclose:

  • What data you collect (Instagram username, DM content, timestamps)
  • Why you collect it (to manage and respond to business inquiries)
  • Where it is stored (Supabase — region you selected)
  • Who can access it (only you, the account owner)
  • How long you retain it and how users can request deletion

Free generators: termly.io, freeprivacypolicy.com, iubenda.com.
